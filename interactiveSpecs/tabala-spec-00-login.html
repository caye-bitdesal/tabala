Section 0 — Login
Entry screen of the app. No username or password fields. Authentication is handled exclusively via Sign in with Google or Autenticació biomètrica (fingerprint / face ID). Both methods produce the same result: a logged-in session.

Screen layout — elements in order
Login screen structure
Background: soft lavender-grey (#F2F0F8). No top bar.
App icon
100×100px rounded square. Yellow background (#FFD000) with the Muixeranga casteller figure in dark red. Centred at the top third of the screen.
App name
"Tabala" in large bold red text (38px). Below the icon.
Subtitle
"Iniciar sessió" in medium grey. Directly below the app name.
Auth card
White rounded card centred vertically. Contains the two auth buttons and the divider between them.
Legal footer
"En continuar, acceptes els termes i condicions de Tabala". Small grey text. Tappable link to T&C.
Authentication methods
G
Sign in with Google
Standard Google button design (Google logo + "Sign in with Google" label). White background, grey border. Triggers OAuth 2.0 Google Sign-In flow via the Android Google Identity Services SDK. On success → server creates or retrieves the user account and issues a session token.
👆
Autenticació biomètrica
White button, red border and red text + fingerprint icon. Uses the Android BiometricPrompt API. Supports fingerprint and face recognition depending on device capability. Only available if the user has previously logged in with Google (biometric is linked to an existing session).
÷
Divider
Horizontal line with "o" (or) centred between the two buttons. Grey colour. Purely visual — no interaction.
Screen states
Idle
Default state. Both buttons active and tappable. No loading indicators.
Biometric prompt
System-level BiometricPrompt dialog overlays the screen. Pulsing fingerprint animation.
Success
Green check + "Sessió iniciada". Auto-redirects to home screen after ~1s.
Loading state
After tapping Google: button shows a spinner and becomes disabled. The rest of the screen is unaffected.
Error state
A snackbar toast appears at the bottom: e.g. "Error en iniciar sessió. Torna-ho a intentar." Buttons re-enable.
Auto-login
If the user has a valid session token, the login screen is skipped entirely and the app launches directly to the home screen.
Biometric availability rules
First-time user
Biometric button is hidden. User must log in with Google first. After a successful Google login, biometric is offered as an option for future logins.
Returning user
Biometric button is visible. Tied to the most recent Google account used on this device.
No biometrics enrolled
If the device has no biometrics set up, the button is hidden. Only Google sign-in is shown.
Biometric changed
If the device detects a new fingerprint or face after login, the existing biometric token is invalidated. User must re-authenticate with Google.
Session expiry
If the server session has expired, biometric re-authentication refreshes the token silently without redirecting to the login screen.
No credentials form — rationale
No username / password
The app deliberately avoids a traditional credentials form. Identity is fully delegated to Google OAuth 2.0, removing password management, recovery flows and security risks for Tabala.
Account creation
There is no separate sign-up screen. Logging in with Google for the first time creates the account automatically. The Mestre then assigns the user to a group.
Account recovery
Not handled by Tabala. Password recovery is handled entirely by Google. Users regain access by recovering their Google account.
Edge cases
Behaviour rules for boundary scenarios.
No internet connection
Google button disabled with a toast: "Sense connexió a internet." Biometric button remains available if a local token exists.
Google account cancelled
User dismisses the Google account picker. Returns silently to the idle login screen. No error shown.
Biometric cancelled
User cancels or fails 3×. Returns to the idle login screen. Toast: "Autenticació cancel·lada."
Biometric lockout
After too many failures, Android locks biometric. Biometric button becomes disabled with a label: "Bloquejat. Usa Google per entrar."
User not yet in a group
Login succeeds but the home screen shows an empty state: "Ets nou a Tabala. El teu mestre t'assignarà a un grup." No error, no crash.
Multiple Google accounts
Android's account picker is shown. User selects the desired account. The last-used account is pre-selected on future logins.
