Section 5 — Figure Planner
Mestre selects a template then assigns members. The figure screen has two diagram views togglable via a button: casteller (front elevation, floor by floor) and aerial (top-down / "planta"). Below the diagram: observations, summary, role legend, and the available persons list — all in one continuous scroll.

El meu perfil
Try it: in "Figure view", toggle between Vista Casteller (floor-by-floor elevation) and Vista Aèria (top-down). Scroll down to see observations, summary, legend and available persons — all in one continuous section.
Mestre flow
Member view
Template list — card fields
Template name
Bold title + short description subtitle.
Role chips
Colour-coded pills with count per role. Same colours used system-wide in diagrams and legends.
Position count
Total positions + number of observations for this template.
Managed by
App administrator. Mestre selects templates but cannot create or modify them.
Diagram view toggle
Two views, one button — togglable above the diagram
The toggle persists per session; the last view selected remains active when scrolling
Vista Casteller
Front elevation view. Shows the figure floor by floor as you would see it from the front. Each person is a coloured block with their name, stacked by floor (Base/Pinya → 1r pis → 2n pis → … → Àngel).
Vista Aèria
Top-down plan view ("planta"). Shows spatial arrangement as seen from above. "MIRA CAP ACÍ ↑" label marks the front direction. Useful for understanding position relative to others.
Toggle placement
Segmented control (pill-style) placed between the top bar and the diagram. Sticky — does not scroll away.
Figure screen — content order (single scroll)
1
Diagram view toggle
Segmented button: "Vista Casteller" | "Vista Aèria". Fixed below the top bar.
2
Diagram (casteller or aerial)
Switches instantly on toggle tap. Scroll position resets to top of diagram on switch.
3
Observations (OBSERVACIONS)
Red bulleted notes from the template. Read-only for members.
4
Summary (RESUM)
4-cell grid: Bases · Alçador/a · Mans · Xiquet. Auto-calculated from template.
5
Role legend (Llegenda de rols)
Colour swatch + role name. Always shown to decode diagram colours.
6
Available persons — Mestre only
Scrollable draggable list continuing below the legend. Count shows assigned/total. Not a separate section — continues the same screen after the legend.
Role colour system
Bases (hot pink)
Alçador/a (cyan)
Xicalla (purple)
Contrafort (orange)
Forçuts (dark red)
Mà — canta (green)
Consistency
Same colour per role in both diagram views, legend chips, template list chips and summary cells.
Unassigned blocks
~50% opacity in both views until a person is assigned.
Print
Printer icon in top bar. Exports the currently active diagram view as a printable PDF.
Member view uses the same screen structure — both diagram views togglable, observations, summary and legend — with two differences: the available persons list is hidden, and the member's own assigned block is highlighted (brighter border/glow) in both views.
What a member sees
Both diagram views
Vista Casteller and Vista Aèria both available. Toggle works identically.
Own position highlight
Member's block has a highlighted border/glow in both views. Helps identify their spot instantly.
Observations
Same red text notes. Useful rehearsal context.
Summary & legend
Same 4-cell grid and colour legend. Decode role colours.
Available persons
Not shown. The persons list section is hidden from members.
Notification
Push on assignment. Badge on Figures tab. Notification tap deep-links directly to the figure.
Edge cases
Behaviour rules for boundary scenarios.
Member RSVP = No but assigned
Mestre can still assign. ⚠️ indicator on their block in Mestre view only.
Member removed from group
Auto-removed from all positions. Suplent 1 becomes titular. Mestre notified.
Publish with empty positions
Allowed. Empty blocks shown at ~50% opacity. No notification for unassigned positions.
Event cancelled after publish
All associated figures deleted. Assigned members receive a cancellation push.
Edit after publish
Mestre can keep editing. Each change triggers a new push to affected members.
Print view
The currently active diagram view (casteller or aerial) is used for the PDF export. Mestre can switch view before printing.
