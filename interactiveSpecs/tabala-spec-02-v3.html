Try it: toggle between Attended and Not Attended to see the gallery access difference.
Automatic attendance registration
✓ Attended
RSVP was Confirmed. "Afegir" button visible in gallery. Counted in past event stats.
✕ Not attended
RSVP ≠ Confirmed or no response. Can view gallery, cannot upload photos.
Trigger
Registered automatically when the event date/time passes. No manual action required.
Rule
RSVP = Confirmed → attended. No / no response → not attended. Final, not correctable.
Mestre exception
Mestre can add photos regardless of their own RSVP status.
Photo upload flow
1
Tap "Afegir"
Visible only to members with attendance registered. Button appears in gallery header.
2
Native Android PhotoPicker
Multi-select. Max 10 MB per photo. No cap on number of photos per session.
3
Preview & confirm
Thumbnails shown. "Pujar [N] fotos" confirm button. Upload runs in background.
4
Direct publish
Photos appear instantly. No moderation queue. Group members receive a push notification.
Gallery behaviour
Layout
2-column grid, square 1:1 thumbnails, rounded corners.
Ordering
Most recently added first.
Viewer
Tap → full-screen. Swipe left/right to navigate. Shows uploader name + upload date.
Download
Download to device available from viewer for all members.
Delete
Members can delete own photos. Mestre can delete any photo.
Edge cases
Behaviour rules for boundary scenarios.
No photos yet
Empty state + "No photos." If attended: "Be the first to add photos" CTA shown.
Photo over 10 MB
Error per oversized file. Other selected photos continue uploading normally.
Upload failure
Auto-retry ×3. Toast with "Retry" if it persists. Successful uploads unaffected.
App closed at transition
Server registers attendance. On next open, event shows as past with attendance already set.
